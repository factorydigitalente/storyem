Before generating the CSV, you MUST internally perform the following three checks. You are STRICTLY PROHIBITED from generating any output until you have completed these verifications and confirmed there are no logical contradictions.

1. Reachability: Is every 'id' reachable from 'id:1' through the choices provided in 'options'? Ensure there are no isolated nodes or "orphan" paragraphs.
2. Flag Consistency: Are items or gold acquired via 'effect' properly utilized or required in subsequent 'options' using [condition] checks?
3. Deadlock Prevention: In paragraphs with conditional choices, ensure that a player who does not meet those conditions will NOT be left with "zero available choices."
4. Guaranteed Acquisition: If an item or gold is required as a [condition] in an option, you MUST ensure there is a physical paragraph where the player can actually acquire it via the effect column. Do not create a "Search" option that leads to a paragraph where no item is actually given (e.g., if the player needs a 'key', one paragraph MUST contain item+key).
5. Resource Loop Verification: If a mandatory item must be purchased at a shop, you MUST ensure there is a repeatable way to earn the required gold (e.g., via a repeatable battle or exploration with !gold+50). It is strictly prohibited to create a situation where a player cannot reach the required amount due to one-time-only rewards, resulting in a literal dead-end.

You must mentally simulate the entire game flow, check the structure for any flaws, and correct them before finalizing the data.
Output the CSV only after ensuring it has passed all internal quality and logic checks.

# Scenario CSV Output Rules
id, type, text, options, e_name, e_skill, e_hp, next, bgm, bgi, effect

▼ Types (type)
- normal: Standard scenes
- battle: Combat scenes
- end: Ending scenes

▼ Options & Condition Checks
Format: [Condition]Display Text→TargetID
1. Gold Check: [gold:200] (Requires 200G or more)
2. Item Requirement: [item_id] (Requires the item)
3. Absence Check: [!item_id] (Only appears if NOT holding the item)
Example: [gold:300]Buy Sword→21

▼ Special Effects (effect)
- Stats: hp+3, hp-2, skill+1, luck-1, food+1
- Items: item+item_id
- Gold: gold+100, gold-50
- Eating: eat (One-time), !eat (Repeatable)
* Use "!" prefix for repeatable effects (e.g., !hp-2).
* Combine with commas (e.g., gold-300,item+iron_sword,skill+3).

▼ Important Formatting
- Always use the full-width arrow "→" in options.
- NO line breaks inside text columns.
- Ensure every row has the same number of commas.

★ Common Mistakes to Avoid
- Missing Commas: The system will not function if the column count is incorrect. Every row MUST have the exact same number of columns (the same number of commas), even if the last column is empty.
- Duplicate IDs: Do not use the same ID more than once.
- Broken Links: Do not point options to an ID that does not exist.
- Arrow Symbol: You MUST use the full-width arrow "→" in the options column. Symbols like "->" or "=>" are strictly prohibited.
- Battle Row Consistency: For "battle" types, columns 5 (Enemy Name) through 8 (Next ID) must be fully populated.
- Quote Management: The "text", "options", and "effect" columns MUST be enclosed in half-width double quotation marks "". The use of double quotation marks (") inside the text is strictly prohibited as it will break the CSV structure. For dialogue or quotes, always use single quotation marks (').

- Item Acquisition & Condition Split: Do not include both "item acquisition (effect)" and "a choice requiring that item (options)" in the same ID. Split them into two paragraphs.
Bad: 6,normal,"Got a key","[key]Open→7",,,,,,,item+key (The button may not work)
Good: 6,normal,"Got a key","Next→61",,,,,,,item+key
61,normal,"You have the key.","[key]Open→7",,,,,,,

- Prevent Deadlocks: When using conditions [condition] in options, ensure the player never gets stuck with "zero available choices." Either perform the condition check *before* moving to the next paragraph, or always provide an unconditional "Go Back" option.
Bad: 2,normal,"At the castle","Next→3",,,,,,,
3,normal,"A locked door","[key]Open→4",,,,,,, (Stuck if no key)
Good: 2,normal,"At the castle","[key]To the door→3,Look for key→5",,,,,,, (Pre-check)
Or: 3,normal,"A locked door","[key]Open→4,Go back→2",,,,,,, (Provides an exit)

★ Additional Guidelines
- Pacing: Keep combat encounters to a minimum. A ratio of about 2 battles per 10 paragraphs provides the best flow.
- Recovery: It is recommended to include recovery points.
- Balance: Placing the "eat" effect before or after a battle helps balance the difficulty.
- Theme: Please select the most appropriate "theme" number (1-9) based on the setting and atmosphere of the story. If you are unsure, use "1".
